# Editing Sounds of Saving (for collaborators)

You edit the site in **Pages CMS** — go to
[app.pagescms.org](https://app.pagescms.org) (or `/admin` on the live site) and
sign in with GitHub.

> **Saving = publishing.** A save commits to the repo and goes live within a
> couple of minutes. There is no draft mode, so finish a change before you save.

## Songs (Songs That Found Me at the Right Time)

Each episode is a form:

- **Artist** and **Song title** — used to build the page web address. Renaming a
  published episode changes its link, so avoid it once shared.
- **Date** — `YYYY-MM-DD`.
- **YouTube URL / ID** — the video. The page embeds it and uses its thumbnail for
  link previews.
- **Featured** — toggle on for the few episodes that should lead the Songs index
  and appear on the home page.
- **Pull quote / Story** — optional editorial text shown under the video.

## News

Each post is a form:

- **Title** — required; builds the web address. Avoid renaming a published post.
- **Date** — required, `YYYY-MM-DD`.
- **Author** — optional ("By ..." byline).
- **Image** — the hero and the link-preview image.
- **Summary** — one line used for search and link previews.
- **Body** — the article (Markdown).

## Pages (Programs, About, Community, Get Help)

Each page is a structured form of headings, body text, and lists (program
pillars, board members, partner logos, form fields). Edit the text in place;
drag list rows to reorder.

## Get Help directory (resources)

The 504-item resource directory is **generated by a script**
(`scripts/parse-resources.js`) from the source data. You can edit
`content/resources.json` in the CMS, but re-running the parser will overwrite
manual changes. For lasting edits, change the parser or the upstream source.

## Site settings

One screen for sitewide text and links: site title, tagline, hero copy, nav,
the Donate URL, the 988 crisis line, social links, and the per-page SEO
descriptions.
